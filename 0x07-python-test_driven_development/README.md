
Test Driven Development (TDD) Python Project

Introduction:
This project follows the Test Driven Development (TDD) approach to software development
where tests are written before the code implementation. 
This README file provides an overview of the project structure, setup instructions
and guidelines for contributing.

Project Structure:

/tests: This directory contains all the test files written for the project.
/src: This directory contains the source code files.
README.md: This file provides information about the project.
LICENSE: The license file for the project.
requirements.txt: Lists all the dependencies required to run the project.
